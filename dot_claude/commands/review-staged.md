# ステージされた変更のレビュー

まず `git diff --cached` をBashツールで実行してください。
diff出力が空の場合は「ステージされた変更がありません」と報告して終了してください。

diff出力をもとに、ステージされた変更をレビューしてください。

1. 変更の要約（ファイル、追加/削除行数、目的）
2. 変更対象ファイルを読み、diffの変更がファイル全体の文脈で正しいか確認
3. 正確性の確認：ロジックエラー、エッジケースの漏れ、タイポ
4. 一貫性の確認：diff内に現れるパターン（命名、修飾子、スタイル等）をGrepで既存コードと照合し、不整合がないか確認
5. 問題があれば指摘し、なければ「問題なし」と記載
